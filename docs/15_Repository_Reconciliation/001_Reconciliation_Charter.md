# REC-001 — Reconciliation Charter

## Purpose

Restore one coherent source of truth across `phoenix-core` and `phoenix-docs`, correct bounded Identity Slice 1 defects, and prevent recurrence through executable integrity gates.

## Principles

Evidence over assertion; no silent scope expansion; security and privacy by design; reversible application deployment; durable data migration; explicit residual risk; synchronized metadata and traceability.

## Acceptance

Repository versions, releases, README files, indexes, manifests, checksums, traceability, governance, tests, migrations, and CI must agree. Both GitHub workflows must pass before verification.
