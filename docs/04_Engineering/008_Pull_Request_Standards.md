# PEH-008 — Pull Request Standards

**Document ID:** PEH-008  
**Knowledge ID:** KN-ENG-008  
**Version:** 1.0  
**Status:** Approved Baseline

## Purpose

A pull request is a reviewable engineering argument, not merely a delivery mechanism.

## Required Content

- problem and intended outcome;
- governing requirement or ADR;
- impacted domains;
- security and privacy impact;
- data migration and rollback;
- test evidence;
- unresolved risks;
- deployment notes.

## Author Responsibilities

- self-review the diff;
- remove noise;
- respond to comments;
- keep documentation current;
- confirm checks pass;
- mark assumptions clearly.

## Merge Conditions

- required checks pass;
- required approvals exist;
- conflicts are resolved;
- migration and rollback are known;
- no unresolved blocking comment remains.
