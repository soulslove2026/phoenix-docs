# PEH-011 — Documentation Standards

**Document ID:** PEH-011  
**Knowledge ID:** KN-ENG-011  
**Version:** 1.0  
**Status:** Approved Baseline

## Principle

Documentation is part of the product and part of implementation.

## Required Metadata

Every governing document includes document ID, version, status, owner, and related documents.

## Writing Rules

- use canonical vocabulary;
- separate requirements from examples;
- label assumptions;
- state scope and non-scope;
- define acceptance criteria;
- avoid unsupported precision;
- link rather than duplicate large sections.

## Change Rules

Behavior changes must update relevant requirements, contracts, diagrams, operations guidance, changelog, and release notes.

## Source of Truth

Committed repository files are authoritative. Conversation notes are not authoritative until committed.
