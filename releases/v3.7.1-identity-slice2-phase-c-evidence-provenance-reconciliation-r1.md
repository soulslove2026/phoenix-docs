# Phoenix Identity Slice 2 Phase C v3.7.1 — External Evidence Provenance Reconciliation Release 1

**Status:** Candidate  
**Production ready:** No  
**Core version:** `3.7.1`  
**Documentation version:** `3.7.1-identity-slice2-phase-c-evidence-provenance-reconciliation-r1`

## Finding

A real-device Windows Hello Passkey registration and authentication exercise succeeded in `local-compose`. The v3.7.0 evidence schema did not admit a local environment, creating pressure to label the record as `staging` merely to pass structural validation.

That label would not reflect the actual execution environment.

## Correction

- Added `local` as a truthful evidence environment.
- Kept local evidence structurally valid and capable of recording a passed engineering exercise.
- Added `nonQualifyingKinds` to the assessment so passed records in an unauthorized environment cannot close Phase C.
- Defined an explicit closure-environment matrix for all eight evidence kinds.
- Corrected recovery-drill templates to use `recovery`, not `staging`.
- Preserved Candidate status and `production_ready=false`.

## Current Passkey conclusion

The Windows Hello exercise proves that Phoenix Passkey registration and authentication work on the tested local Windows/Chromium platform. It does not prove staging or production behavior and therefore does not close ID2-038.

## Next gate

Deploy a governed HTTPS staging environment with independent secrets, database, observability, and WebAuthn RP/origin configuration. Repeat the real-device registration and authentication exercise there and collect sanitized evidence.
