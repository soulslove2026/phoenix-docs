# Phoenix Identity Slice 2 Phase C v3.8.0 — Staging Assurance Foundation Release 1

**Status:** Candidate  
**Date:** 2026-07-15  
**Production ready:** No  
**Core version:** `3.8.0`  
**Documentation version:** `3.8.0-identity-slice2-phase-c-staging-assurance-foundation-r1`

## Purpose

Create a governed, provider-neutral staging baseline that can host qualifying real-world assurance exercises without misrepresenting local or CI evidence as staging evidence.

## Added

- fail-closed staging runtime contract;
- required deployment identity, region, and source commit;
- mandatory TLS proxy posture, database, and protected operations monitoring;
- non-local WebAuthn RP ID and HTTPS-only origins;
- secret-file loading through mutually exclusive `_FILE` inputs;
- provider-neutral staging Compose topology with private data network;
- sanitized staging preflight and remote smoke tooling;
- dedicated Staging Foundation Validation workflow;
- staging deployment and Passkey ceremony runbooks.

## Boundary

This release does not select a cloud provider, deploy a domain, perform a real staging Passkey ceremony, or close any external-assurance record. Phase C remains Candidate and production readiness remains false.

## Next gate

Deploy the verified immutable image behind a real HTTPS staging origin, run preflight and smoke validation, complete Passkey registration and authentication, sanitize the evidence, and validate it as `environment: staging`.
