# Phoenix Architecture Foundation v2.0.0 — Release 1

**Release identifier:** `2.0.0-architecture-r1`  
**Date:** 2026-07-13  
**Status:** Ratified Foundation Specification

## Purpose

Establish the first authoritative architecture foundation for Phoenix through architecture vision, bounded contexts, domain relationships, capability mapping, and system landscape.

## Added

- ARC-000 Architecture Foundation Read First
- ARC-001 Architecture Vision
- ARC-002 Bounded Contexts
- ARC-003 Domain Map
- ARC-004 Capability Map
- ARC-005 System Landscape

## Updated

README, Master Index, Document Index, Traceability Matrix, Project Board, Engineering Roadmap, Architect Notes, release metadata, manifest, integrity report, and checksums.

## Major Decisions

- Bounded-context modularity is the initial architecture model.
- Microservices are selective extractions, not the default starting point.
- Critical contexts receive risk-weighted isolation.
- Domain truth remains separate from projections, search, analytics, and AI.
- External providers are integrated through controlled adapters and reconciliation.
